# 弹幕属性

| 属性               | 类型      | 说明                                                         |
| ------------------ | --------- | ------------------------------------------------------------ |
| `uname`            | `string`  | 弹幕发送者的用户名称                                         |
| `uid`              | `number`  | 弹幕发送者的 UID                                             |
| `danmaku`          | `string`  | 弹幕内容                                                     |
| `type`             | `number`  | 0: 文本弹幕 / 文本+普通表情弹幕<br />1: 表情弹幕             |
| `show_reply`       | `boolean` | 默认为 `true`, *未知改变条件* <br />自己发送的弹幕为 `false` |
| `replymid`         | `number`  | 引用对象的UID<br />如果当前弹幕 @ 某个人, 该字段为对应用户的 UID |
| `ts`               | `number`  | 1. 自己发送的弹幕:  `ts` 为 0 <br />2. 其它弹幕: `ts` 为本机接受到该弹幕时的时间 (秒级时间戳) |
| `timestamp`        | `number`  | 1. 用户 (*你*) 进入直播间时, 默认显示几条弹幕: 这些弹幕的 `timestamp` 为 *你* 进入直播间的时间 (秒级时间戳)<br />2. 新接受到的弹幕: `timestamp` 为该弹幕发送到 Bilibili 服务器的时间 (秒级时间戳)<br />3. 自己发送的弹幕: `timestamp` 为弹幕发送时的时间 (毫秒级时间戳) |
| `ct`               | `number`  | *未知*<br />自己发送的弹幕为空字符串                         |
| `score`            | `number`  | *未知*<br />自己发送的弹幕为 0                               |
| `isunilivedanmaku` | `boolean` | *未知*                                                       |
| `originAnchorName` | `string`  | *未知*                                                       |
| `originRoomInfo`   | `string`  | *未知*                                                       |
| `sourceRoomId`     | `number`  | *未知*                                                       |
| `id_str`           | `string`  | *未知*                                                       |

> *自己发送的弹幕* 指从弹幕栏点击发送后, 显示的弹幕. 
> 这些弹幕没有经过服务器就直接显示在弹幕栏, 所以会存在你发送了弹幕显示在弹幕框上, 但是因为违禁处理其实并没有发送的情况. 
>
> 重新刷新后显示的自己发送的弹幕, 不算 *自己发送的弹幕* 这些规则, 和其它正常显示的弹幕一样.
